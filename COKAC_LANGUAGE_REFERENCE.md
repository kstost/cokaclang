# COKAC 언어 레퍼런스 (실사용 상세판)

이 문서는 현재 구현 기준(`cokaclang`, Rust)의 문법/런타임/내장 함수를 실사용 관점에서 설명합니다.
핵심 목표는 다음 두 가지입니다.

- 문법을 정확히 이해하고 바로 코드를 작성할 수 있다.
- 내장 함수의 사용법(인자/반환/예시)을 보고 바로 사용할 수 있다.

---

## 1) 시작

### 빌드
```bash
python3 build.py
```
디버그 빌드:
```bash
python3 build.py --debug
```

### 실행
```bash
./cokaclang 파일.cokac
./cokaclang 파일.cokac 인수1 인수2
```

### 테스트
```bash
./cokaclang --test tests
./cokaclang --test-json tests
```

---

## 2) CLI

- `./cokaclang <file.cokac> [인수...]`
- `./cokaclang --test <파일|디렉토리> [추가...]`
- `./cokaclang --test-json <파일|디렉토리> [추가...]`

추가 동작 규칙:
- `--test`/`--test-json`은 대상 경로에서 `.cokac` 파일만 수집해 실행합니다.
- 디렉토리 내 보조 모듈/픽스처는 `.mod` 같은 확장자를 사용하면 테스트 대상에서 제외할 수 있습니다.
- 테스트 실행 시 각 테스트 파일은 별도 프로세스로 실행되며, 한 파일의 전역 상태는 다른 파일에 공유되지 않습니다.
- 종료코드:
  - 전체 통과: `0`
  - 하나 이상 실패: `1`

## 3) 문법 핵심

### 주석
- 한 줄: `// ...`
- 블록: `/* ... */`

### 키워드
- 선언/제어: `변수`, `상수`, `출력`, `만약`, `아니면`, `반복`, `동안`, `중단`, `계속`
- 함수/비동기: `함수`, `반환`, `비동기`, `대기`
- 모듈/예외: `가져오기`, `별칭`, `시도`, `잡기`, `마침`, `던지기`
- 리터럴/논리: `참`, `거짓`, `없음`, `그리고`, `또는`

### 값 타입
- `숫자`, `불린`, `문자열`, `배열`, `객체`, `함수`, `작업`, `없음`
- 타입 확인: `타입(x)`

### 연산자
- 산술: `+ - * / %`
- 비교: `== != < <= > >=`
- 논리: `그리고 또는 !`

### 접근
- 함수 호출: `f(a, b)`
- 객체 속성: `obj.키`
- 인덱스: `arr[0]`, `obj["키"]`
- 체인: `함수().프로퍼티`

### 제어문
```cokac
변수 x = 3;
만약 (x > 0) { 출력 "양수"; } 아니면 { 출력 "0이하"; }

반복 (x > 0) { x = x - 1; }

동안 (변수 i = 0; i < 10; i = i + 1) {
    만약 (i == 3) { 계속; }
    만약 (i == 8) { 중단; }
}
```

### 함수
```cokac
함수 더하기(a, b) { 반환 a + b; }
출력 더하기(2, 3);
```

### 예외
```cokac
시도 {
    던지기 "실패";
} 잡기(메시지, 오류정보) {
    출력 메시지;
    출력 오류정보.코드;
} 마침 {
    출력 "정리";
}
```

### 모듈
```cokac
가져오기 "모듈.cokac" 별칭 M;
출력 M.함수();

내보내기("값", 123);
변수 모듈 = 모듈가져오기("모듈.cokac");
출력 모듈.값;
```

모듈 내장 함수:

- `내보내기(이름, 값)`
반환: `없음`
예시:
```cokac
함수 합(a, b) { 반환 a + b; }
내보내기("합", 합);
내보내기("버전", "1.0.0");
```

- `모듈가져오기(경로)`
반환: `객체` (해당 모듈의 export 집합)
예시:
```cokac
변수 M = 모듈가져오기("수학모듈.cokac");
출력 M.버전;
출력 M.합(3, 4);
```

### 3.A 형식 문법/의미론 상세 (구현 기준)

이 절은 문법 핵심(3장)을 실구현 동작까지 포함해 더 엄밀하게 기술한 규격입니다.

#### 3.A.1 토큰/어휘 규칙
- 식별자:
  - 시작 문자: 유니코드 문자(`is_alphabetic`), `_`, 또는 비ASCII 문자
  - 이후 문자: 유니코드 문자/숫자(`is_alphanumeric`), `_`, 또는 비ASCII 문자
- 숫자 리터럴:
  - 리터럴 자체는 `DIGIT+ ("." DIGIT+)?`
  - 음수는 단항 `-`로 해석 (`-3` == `-(3)`)
  - 숫자 변환은 `숫자(x)` 내장 함수를 사용
- 문자열 리터럴:
  - 큰따옴표(`"..."`) 사용
  - 지원 이스케이프: `\\n`, `\\r`, `\\t`, `\\\\`, `\\\"`
  - `\uXXXX`, `\/`, `\b`, `\f`는 현재 문자열 리터럴에서 지원하지 않음
- 주석:
  - `#` 한 줄
  - `//` 한 줄
  - `/* ... */` 블록

#### 3.A.2 문장 종료와 블록
- 문장(statement)은 기본적으로 세미콜론(`;`)으로 종료합니다.
- 블록은 `{ ... }`로 표현합니다.
- 함수/조건/반복/예외 블록 내부에서도 동일 규칙을 유지합니다.

#### 3.A.3 연산자 우선순위/결합성
높은 우선순위에서 낮은 우선순위 순:
1. 호출/속성/인덱스: `f(...)`, `a.b`, `a[b]`
2. 단항: `!`, `-`, `대기`
3. 곱셈군: `*`, `/`, `%`
4. 덧셈군: `+`, `-`
5. 비교: `<`, `<=`, `>`, `>=`
6. 동등: `==`, `!=`
7. 논리곱: `그리고`
8. 논리합: `또는`
9. 대입: `=` (문장 레벨)

현재 구현에서 지원되는 대입 연산자는 `=`만입니다.
`+=`, `-=`, `*=`, `/=`는 토큰/문서 표기 대비 미구현 상태입니다.

#### 3.A.4 대입의 좌변 제약
- 유효한 좌변:
  - 변수 이름: `x = ...`
  - 인덱스: `arr[i] = ...`, `obj["k"] = ...`
  - 속성: `obj.k = ...`
- 그 외 식은 대입 좌변으로 사용할 수 없습니다.

#### 3.A.5 제어문 의미론
- `만약/아니면`: 조건식 truthy 기준 분기
- `반복`: while 스타일 (`조건`이 거짓이 될 때까지)
- `동안`: for 스타일 (초기화/조건/증감)
- `중단`: 가장 가까운 반복문 탈출
- `계속`: 가장 가까운 반복문의 다음 반복으로 이동

구현 제약:
- 반복문 바깥 `중단`/`계속`은 오류입니다.

#### 3.A.6 함수 선언/호출 의미론
- 함수는 1급 값입니다.
- 사용자 함수 인자 개수 불일치:
  - 부족한 인자: `없음`으로 채움
  - 초과한 인자: 무시
- 내장 함수는 각 함수가 인자 개수/타입을 개별 검증합니다.
- `반환`을 만나면 즉시 함수 호출을 종료하고 값을 반환합니다.

구현 제약:
- 함수 바깥 `반환`은 오류입니다.

#### 3.A.7 스코프와 이름 해석
- 렉시컬 스코프 기반 환경 체인을 사용합니다.
- 조회 순서: `현재 스코프 -> 부모 스코프 ...`
- `상수`는 재할당 불가입니다.
- `상수`에 할당된 배열/객체는 불변 처리됩니다.

구현 제약:
- 동일 스코프에서 같은 이름 `변수` 재선언은 오류입니다.
- 반복문 내부에서 동일 이름을 반복마다 재선언하는 패턴은 오류를 유발할 수 있으므로,
  루프 바깥 1회 선언 + 내부 재할당 패턴을 권장합니다.

#### 3.A.8 클로저 의미론
- 함수는 선언 시점의 렉시컬 환경을 캡처합니다.
- 반환된 클로저는 캡처된 상태를 호출 간 유지합니다.
- 서로 다른 클로저 인스턴스의 상태는 독립이어야 합니다.

#### 3.A.9 예외 의미론
- `던지기 x`는 현재 실행을 중단하고 예외를 전파합니다.
- `시도/잡기/마침`:
  - `시도`에서 오류 발생 시 `잡기`가 있으면 처리
  - `마침`은 성공/실패와 관계없이 실행
- `잡기(메시지, 오류정보)`의 `오류정보`에는 코드/줄/스택이 포함됩니다.

#### 3.A.10 모듈 의미론
- `가져오기 "path" 별칭 M`: 모듈 export를 별칭 객체로 바인딩
- `모듈가져오기("path")`: export 객체를 값으로 반환
- `내보내기(k, v)`: 모듈 공개 API 등록
- 모듈은 캐시되며 동일 경로 재로딩 시 캐시 객체를 재사용합니다.

구현 제약:
- 모듈 최상위 `반환`/`중단`/`계속`은 오류입니다.
- 상대 경로 해석은 `current_file`(현재 실행 스크립트/현재 모듈 파일)의 부모 디렉토리 기준입니다.
- 가능하면 canonicalize(정규 절대 경로) 후 캐시 키로 사용합니다.

#### 3.A.11 비동기 의미론
- `비동기 함수` 호출 결과는 `작업(Task)`입니다.
- `대기 task`/`대기최대(task, ms)`로 동기화합니다.
- `작업모두([task...])`: 전부 완료 시 배열 결과
- `작업경주([task...])`: 먼저 완료(성공/실패)한 작업 결과/오류를 반영
- `작업취소(task)`: 미완료 작업을 취소 시도

#### 3.A.12 값 동등성
- 기본 타입은 값 기준 비교
- 배열/객체는 구조적 비교를 수행
- 작업(Task)은 상태/결과 기반 비교 경로를 가집니다
- 숫자 동등 비교는 `abs(a-b) < 1e-9` 허용 오차를 사용합니다.

#### 3.A.13 내장 함수 호출 규칙
- 내장 함수는 인자 개수/타입 검증을 엄격히 수행합니다.
- 인자 불일치 시 명시적 오류를 반환합니다.
- 운영 코드는 예외 메시지보다 `오류정보.코드` 기반 분기를 권장합니다.

#### 3.A.14 표현식 평가 순서
- 함수 호출 인자는 좌 -> 우 순서로 평가됩니다.
- 단락 평가(short-circuit):
  - `그리고`: 좌항이 거짓이면 우항 미평가
  - `또는`: 좌항이 참이면 우항 미평가

#### 3.A.15 최소 문법 준수 체크리스트
1. 함수 바깥 `반환` 사용 금지
2. 반복문 바깥 `중단/계속` 사용 금지
3. 동일 스코프 변수 재선언 금지
4. 모듈 최상위 제어문(`반환/중단/계속`) 금지
5. 테스트 픽스처 파일 확장자 분리(`.mod` 권장)

#### 3.A.16 진릿값/출력/타입명 규약
- Truthy/Falsy:
  - `거짓`, `없음`, 빈 문자열(`""`), 숫자 `0` 근처(`abs(n) <= 1e-9`), 빈 배열, 빈 객체는 falsy
  - 함수는 항상 truthy
  - 작업(Task)은 `완료 && !실패`일 때 truthy
- 출력 문자열:
  - `출력 없음;`의 표시 문자열은 현재 구현에서 `nil`
  - 불리언 출력은 `참`/`거짓`
- `타입(x)` 관련 타입명:
  - 구현 내부 타입명은 `불리언`을 사용합니다(문서의 설명상 `불린`과 동일 의미).

## 4) 비동기 모델

핵심 규칙:
- `비동기 함수`를 호출하면 즉시 `작업(Task)`을 반환합니다.
- 결과가 필요할 때 `대기 task` 또는 `대기최대(task, 밀리초)`를 사용합니다.
- `작업취소(task)`는 아직 완료되지 않은 작업에 취소를 요청합니다.
- `작업모두/작업경주`는 여러 작업을 조합하는 조합기입니다.

실전 패턴:
```cokac
비동기 함수 계산(a, b) {
    반환 a + b;
}

변수 t = 계산(10, 20);          // 즉시 Task 반환
출력 작업상태(t);               // "대기" 또는 "완료" 등
변수 r = 대기최대(t, 3000);     // 3초 내 완료 대기
출력 r;                         // 30
```

작업(Task) 관련 함수:
- 상태 조회: `작업완료`, `작업실패`, `작업상태`, `작업오류`, `작업오류코드`, `작업결과`
- 제어: `작업취소`
- 조합: `작업모두`, `작업경주`
- 런타임 메트릭: `비동기큐길이`, `비동기통계`

`비동기통계()` 반환 객체 키:
- `누적등록`, `누적완료`, `누적실패`, `누적취소`
- `누적재큐`, `누적백프레셔`
- `누적루프틱`, `누적대기호출`, `누적대기타임아웃`
- `최대큐길이`, `현재큐길이`

## 6) 내장 함수 상세 사용법

아래는 내장 함수 전부를 카테고리별로 정리한 사용 레퍼런스입니다.
표기 규칙:

- 형식: `함수명(인자...)`
- 반환: 반환 타입/의미
- 예시: 바로 실행 가능한 한 줄

### 6.1 기본/변환/검증

- `길이(x)`  
  반환: 숫자(길이)  
  예시: `출력 길이("abc");`

- `숫자(x)`  
  반환: 숫자 변환값  
  예시: `출력 숫자("12.5");`

- `불린(x)`  
  반환: 참/거짓  
  예시: `출력 불린(0);`

- `정수(x)`  
  반환: 소수점 버림 숫자  
  예시: `출력 정수(3.9);`

- `문자열(x)`

  예시: `변수 req = 요청받기(s);`

- `비동기요청받기(서버소켓)`  
  반환: 작업(Task) -> `대기` 시 요청 객체  
  예시: `변수 req = 대기 비동기요청받기(s);`

- `응답보내기(연결, 상태코드, 본문 [,헤더객체])`  
  반환: 참/거짓  
  예시: `응답보내기(req.연결, 200, "ok");`

- `연결닫기(소켓)`  
  반환: 참/거짓  
  예시: `연결닫기(req.연결);`

### 6.13 비동기 함수(HTTP/파일/경로/디렉토리)

비동기 함수는 모두 `Task`를 반환합니다. 결과는 `대기`/`대기최대`로 얻습니다.

#### 6.13.1 비동기 HTTP

- `비동기웹요청(method, url [,헤더 [,본문]])`  
  반환: `Task` -> `대기` 시 응답 객체  
  예시: `변수 r = 대기 비동기웹요청("GET", "https://example.com");`

- `비동기가져오기요청(url [,옵션객체])`  
  반환: `Task` -> `대기` 시 응답 객체  
  예시: `변수 r = 대기 비동기가져오기요청("https://example.com", {재시도:2});`

#### 6.13.2 비동기 파일

- `비동기파일읽기(경로)`  
  반환: `Task` -> 문자열
  예시: `변수 내용 = 대기 비동기파일읽기("a.txt");`
- `비동기파일쓰기(경로, 내용)`  
  반환: `Task` -> 참/거짓
  예시: `변수 ok = 대기 비동기파일쓰기("a.txt", "hello");`
- `비동기파일추가(경로, 내용)`  
  반환: `Task` -> 참/거짓
  예시: `대기 비동기파일추가("a.txt", "\\nmore");`
- `비동기파일읽기줄들(경로)`  
  반환: `Task` -> 문자열 배열
  예시: `변수 줄들 = 대기 비동기파일읽기줄들("a.txt");`
- `비동기파일쓰기줄들(경로, 줄배열)`  
  반환: `Task` -> 참/거짓
  예시: `대기 비동기파일쓰기줄들("a.txt", ["a", "b", "c"]);`
- `비동기파일복사(원본, 대상)`  
  반환: `Task` -> 참/거짓
  예시: `대기 비동기파일복사("a.txt", "b.txt");`
- `비동기파일이동(원본, 대상)`  
  반환: `Task` -> 참/거짓
  예시: `대기 비동기파일이동("b.txt", "c.txt");`
- `비동기파일삭제(경로)`  
  반환: `Task` -> 참/거짓
  예시: `변수 deleted = 대기 비동기파일삭제("c.txt");`
- `비동기파일존재(경로)`  
  반환: `Task` -> 참/거짓
  예시: `변수 exists = 대기 비동기파일존재("a.txt");`
- `비동기파일정보(경로)`  
  반환: `Task` -> 객체
  예시: `변수 info = 대기 비동기파일정보("a.txt"); 출력 info.크기;`
- `비동기파일크기(경로)`  
  반환: `Task` -> 숫자
  예시: `출력 대기 비동기파일크기("a.txt");`
- `비동기파일수정시각(경로)`  
  반환: `Task` -> 숫자(Unix epoch)
  예시: `출력 대기 비동기파일수정시각("a.txt");`

예시:
```cokac
변수 t = 비동기파일읽기("a.txt");
변수 내용 = 대기최대(t, 3000);
출력 내용;
```

#### 6.13.3 비동기 디렉토리/경로

- `비동기디렉토리목록([경로])`  
  반환: `Task` -> 배열  
  예시: `변수 목록 = 대기 비동기디렉토리목록(".");`

- `비동기현재디렉토리()`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기현재디렉토리();`

- `비동기디렉토리생성(경로)`  
  반환: `Task` -> 참/거짓  
  예시: `대기 비동기디렉토리생성("tmpdir");`

- `비동기디렉토리삭제(경로)`  
  반환: `Task` -> 참/거짓  
  예시: `대기 비동기디렉토리삭제("tmpdir");`

- `비동기디렉토리삭제재귀(경로)`  
  반환: `Task` -> 참/거짓  
  예시: `대기 비동기디렉토리삭제재귀("tmpdir");`

- `비동기디렉토리복사(원본, 대상)`  
  반환: `Task` -> 참/거짓  
  예시: `대기 비동기디렉토리복사("a", "b");`

- `비동기디렉토리존재(경로)`  
  반환: `Task` -> 참/거짓  
  예시: `출력 대기 비동기디렉토리존재(".");`

- `비동기경로존재(경로)`  
  반환: `Task` -> 참/거짓  
  예시: `출력 대기 비동기경로존재("a.txt");`

- `비동기경로합치기(기준, 하위)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기경로합치기("/tmp", "a.txt");`

- `비동기절대경로(경로)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기절대경로("a.txt");`

- `비동기경로이름(경로)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기경로이름("/tmp/a.txt");`

- `비동기상위경로(경로)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기상위경로("/tmp/a.txt");`

- `비동기확장자(경로)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기확장자("a.txt");`

- `비동기경로정규화(경로)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기경로정규화("./a/../b.txt");`

- `비동기상대경로(기준, 대상)`  
  반환: `Task` -> 문자열  
  예시: `출력 대기 비동기상대경로("/tmp", "/tmp/a/b.txt");`

#### 6.13.4 비동기 서버 수락

- `비동기요청받기(서버소켓)`  
  반환: `Task` -> 요청 객체

예시:
```cokac
변수 서버 = 서버열기(3000, "0.0.0.0", 64);
변수 req = 대기 비동기요청받기(서버);
응답보내기(req.연결, 200, "ok");
연결닫기(req.연결);
```

### 6.14 Task 조합/제어/관측

- `대기(task)`  
  반환: Task 결과, 실패 시 예외
  예시: `변수 r = 대기 t;`

- `대기최대(task, 밀리초)`  
  반환: Task 결과, 시간 초과 시 `E_TIMEOUT`
  예시: `변수 r = 대기최대(t, 3000);`

- `작업완료(task)`  
  반환: 참/거짓
  예시: `출력 작업완료(t);`

- `작업실패(task)`  
  반환: 참/거짓
  예시: `출력 작업실패(t);`

- `작업상태(task)`  
  반환: `"대기" | "완료" | "실패" | "취소"`
  예시: `출력 작업상태(t);`

- `작업오류(task)`  
  반환: 오류 메시지 문자열 또는 `없음`
  예시: `출력 작업오류(t);`

- `작업오류코드(task)`  
  반환: 오류 코드 문자열 또는 `없음`
  예시: `출력 작업오류코드(t);`

- `작업결과(task)`  
  반환: 완료된 작업의 결과 값(미완료면 `없음`)
  예시: `출력 작업결과(t);`

- `작업취소(task)`  
  반환: 취소 요청 성공 시 참
  예시: `출력 작업취소(t);`

- `작업모두(작업배열)`  
  반환: `Task` -> 모든 작업 결과 배열
  예시: `변수 all = 작업모두([a, b, c]);`

- `작업경주(작업배열)`  
  반환: `Task` -> 가장 먼저 끝난 작업 결과
  예시: `변수 first = 작업경주([a, b, c]);`

- `비동기큐길이()`  
  반환: 숫자(현재 큐 길이)
  예시: `출력 비동기큐길이();`

- `비동기통계()`  
  반환: 메트릭 객체
  예시: `변수 m = 비동기통계(); 출력 m.누적완료;`

예시:
```cokac
변수 a = 비동기파일읽기("a.txt");
변수 b = 비동기파일읽기("b.txt");
변수 둘다 = 작업모두([a, b]);
변수 결과들 = 대기최대(둘다, 5000);
출력 길이(결과들);
```

## 9) 언어적 특성 Deep Dive

### 9.1 설계 철학
- COKAC은 `원초적 기능(capability)` 중심 설계를 지향합니다.
- 고수준 편의 기능은 가능한 한 코어가 아닌 함수/모듈 조합으로 쌓는 방향입니다.
- 표면 문법은 한글 중심으로 유지하고, 런타임 의미론은 명시적/예측 가능하게 유지합니다.
- Do: 코어에는 최소 기능만 두고, 도메인 로직은 모듈로 분리하세요.
- Don’t: 편의 함수 하나를 위해 코어 내장 함수를 계속 늘리지 마세요.

### 9.2 문법 성격
- 선언(`변수/상수/함수`)과 명령(`만약/반복/동안/반환/던지기`)이 공존하는 명령형 중심 언어입니다.
- 문장은 세미콜론(`;`) 기반, 블록은 `{ ... }` 기반입니다.
- 식(expression)과 문(statement)이 분리되어 있어, 제어 흐름 해석이 명확합니다.
- Do: 블록 경계를 명확히 하고 문장 끝 세미콜론을 일관되게 유지하세요.
- Don’t: 한 줄에 많은 식을 체인으로 붙여 가독성을 해치지 마세요.

### 9.3 스코프/이름해석
- 렉시컬 스코프(정적 스코프) 기반 환경 체인으로 이름을 해석합니다.
- 조회 순서는 `지역 -> 바깥 -> 전역`입니다.
- `상수`는 재할당 불가이며, 불변 제약과 함께 사용하는 패턴이 권장됩니다.
- Do: 공유 상태는 `상수 + 객체설정`보다 모듈 함수 경유 업데이트를 우선하세요.
- Don’t: 같은 이름을 여러 스코프에서 반복 선언해 그림자 변수(shadowing)를 남발하지 마세요.

### 9.4 데이터 모델
- 핵심 타입: `숫자`, `불린`, `문자열`, `배열`, `객체`, `함수`, `작업`, `없음`.
- 동적 타입 언어이며, 변환은 `숫자/문자열/불린/정수` 같은 명시 함수로 제어합니다.
- 오류코드 기반 모델을 함께 제공해 동적 타입의 운영 리스크를 줄입니다.
- Do: 외부 입력(JSON/HTTP/파일)은 즉시 타입 점검 후 내부 모델로 변환하세요.
- Don’t: 암묵 변환 결과를 가정하고 연산을 이어붙이지 마세요.

### 9.5 함수 모델
- 함수는 1급 값입니다.
- 함수 전달/반환/저장이 가능하고, 콜백/함수 조합 패턴을 구성할 수 있습니다.
- 내장 함수/사용자 함수를 동일한 호출 형태로 다룹니다.
- Do: 작은 순수 함수들을 조합해 테스트 가능한 구조를 만드세요.
- Don’t: 함수 내부에서 전역 상태를 과도하게 읽고 쓰는 구조를 만들지 마세요.

### 9.6 객체/배열 모델
- 객체는 점 접근(`obj.key`)과 인덱스 접근(`obj["key"]`)을 모두 지원합니다.
- 배열은 순차 컨테이너이며 삽입/삭제/슬라이스/정렬 등 조작 함수를 제공합니다.
- 상수와 결합될 때의 불변성 규칙을 고려해 데이터 수정 경계를 설계하는 것이 좋습니다.
- Do: 공개 데이터는 객체 스키마를 문서화하고 키 이름을 고정하세요.
- Don’t: 배열 인덱스 범위를 검증하지 않고 직접 접근하지 마세요.

### 9.7 비동기 모델의 의미론
- `비동기 함수` 호출은 `작업(Task)`을 반환하고 `대기`/`대기최대`로 동기화합니다.
- 취소/실패/오류코드/통계 조회를 제공해 운영 제어가 가능합니다.
- 런타임은 이벤트 대기(epoll) + 작업 큐 + 워커 경로를 혼합한 실용형 모델입니다.
- Do: 외부 I/O는 `대기최대`로 타임아웃을 항상 설정하세요.
- Do: `작업오류코드`를 기준으로 재시도/중단을 분기하세요.
- Don’t: `대기`를 무제한으로 남발해 장애 시 무한 지연을 만들지 마세요.

### 9.8 모듈/네임스페이스 모델
- `가져오기`, `가져오기 ... 별칭`, `내보내기`, `모듈가져오기`를 제공합니다.
- 별칭 import는 네임스페이스처럼 동작하므로 대규모 코드 충돌 방지에 유리합니다.
- 모듈 경계의 공개 API는 `내보내기`로 명시하는 스타일을 권장합니다.
- Do: 항상 `별칭`을 사용해 모듈 경계를 명확히 하세요.
- Don’t: `가져오기`만으로 전역 심볼을 누적시키는 패턴을 장기간 유지하지 마세요.

### 9.9 오류 모델
- 예외 문법: `시도/잡기/마침`, `던지기`.
- `잡기(메시지, 오류정보)`에서 메시지 + 구조화 오류정보(코드/줄/스택)를 함께 처리합니다.
- 운영 자동화에서는 `오류정보.코드`를 기준으로 분기하는 것이 안정적입니다.
- Do: 문자열 메시지보다 `오류정보.코드`를 1차 분기 키로 사용하세요.
- Don’t: 모든 오류를 한 catch에서 삼키고 계속 진행하지 마세요.

### 9.10 실무 관점 요약
- 강점:
  - 한글 중심 문법
  - 풍부한 내장 기능(파일/네트워크/서버/비동기)

## 10) 문법 참조(EBNF 유사 표기)

### 10.1 프로그램/문장
- `program := statement* EOF`
- `statement := block | varDecl | constDecl | functionDecl | ifStmt | whileStmt | forStmt | printStmt | returnStmt | breakStmt | continueStmt | throwStmt | tryStmt | importStmt | assignStmt | exprStmt`
- `block := "{" statement* "}"`
- `assignStmt := IDENT "=" expression ";" | memberAccess "=" expression ";" | indexAccess "=" expression ";"`
- `exprStmt := expression ";"`

### 10.2 선언/제어
- `varDecl := "변수" IDENT "=" expression ";"`
- `constDecl := "상수" IDENT "=" expression ";"`
- `functionDecl := ("비동기")? "함수" IDENT "(" params? ")" block`
- `ifStmt := "만약" "(" expression ")" block ("아니면" (ifStmt | block))?`
- `whileStmt := "반복" "(" expression ")" block`
- `forStmt := "동안" "(" (varDecl | constDecl | exprStmt | ";") expression? ";" forIncrement? ")" block`
- `forIncrement := (IDENT "=" expression) | expression`
- `returnStmt := "반환" expression? ";"`
- `breakStmt := "중단" ";"`
- `continueStmt := "계속" ";"`
- `throwStmt := "던지기" expression ";"`
- `tryStmt := "시도" block ("잡기" "(" IDENT ("," IDENT)? ")" block)? ("마침" block)?`
- 제약: `tryStmt`는 `잡기` 또는 `마침` 중 하나 이상 반드시 포함

### 10.3 식
- `expression := logic_or`
- `logic_or := logic_and ("또는" logic_and)*`
- `logic_and := equality ("그리고" equality)*`
- `equality := comparison (("==" | "!=") comparison)*`
- `comparison := term ((">" | ">=" | "<" | "<=") term)*`
- `term := factor (("+" | "-") factor)*`
- `factor := unary (("*" | "/" | "%") unary)*`
- `unary := ("!" | "-" | "대기") unary | call`
- `call := primary ( "(" args? ")" | "." IDENT | "[" expression "]" )*`
- `primary := NUMBER | STRING | "참" | "거짓" | "없음" | IDENT | "(" expression ")" | arrayLit | objectLit`

### 10.4 리터럴
- `arrayLit := "[" (expression ("," expression)*)? "]"`
- `objectLit := "{" ((IDENT | STRING) ":" expression ("," (IDENT | STRING) ":" expression)*)? "}"`

### 10.5 어휘(LEXICAL) 규칙
- `IDENT := (LETTER | "_" | NON_ASCII) (LETTER | DIGIT | "_" | NON_ASCII)*`
- `LETTER := UnicodeAlphabetic`
- `DIGIT := [0-9]`
- `NON_ASCII := codepoint >= 0x80`
- `NUMBER := DIGIT+ ("." DIGIT+)?`
- `STRING := "\"" (ESC | ~["\\])* "\""`
- `ESC := "\\" ("\"" | "\\" | "n" | "r" | "t")`

키워드 집합:
- 선언/제어: `변수`, `상수`, `출력`, `만약`, `아니면`, `반복`, `동안`, `중단`, `계속`
- 함수/비동기: `함수`, `비동기`, `반환`, `대기`
- 모듈/예외: `가져오기`, `별칭`, `시도`, `잡기`, `마침`, `던지기`
- 리터럴/논리: `참`, `거짓`, `없음`, `그리고`, `또는`

주석:
- `# ...` (한 줄)
- `// ...` (한 줄)
- `/* ... */` (블록)

### 10.6 문맥 제약(CONTEXT RULES)
- `반환`은 함수 본문 내부에서만 유효
- `중단`/`계속`은 반복문 내부에서만 유효
- 모듈 최상위에서 `반환`/`중단`/`계속` 금지
- 동일 스코프에서 같은 이름 재선언 금지
- 상수(`상수`)는 재할당 금지
- 대입 좌변은 `IDENT | memberAccess | indexAccess`만 허용

### 10.7 연산자 결합 규칙(요약)
- 단항: `!`, `-`, `대기`
- 이항(우선순위 낮아짐):
  - `* / %`
  - `+ -`
  - `< <= > >=`
  - `== !=`
  - `그리고`
  - `또는`
- 대입 `=`은 표현식 연산자가 아니라 문장 레벨 대입 구문으로 처리됩니다.

## 12) 문제 해결(FAQ)

### 12.1 `정의되지 않은 변수` 오류
- 원인:
  - 선언 누락
  - 스코프 밖 접근
- 해결:
  - 변수 선언 위치를 호출보다 앞에 두기
  - 모듈은 `별칭`으로 가져와 `별칭.이름` 접근

### 12.2 `E_SECURITY_POLICY`
- 원인:
  - 보안 정책 위반(예: HTTPS 검증 비활성화 시도)
- 해결:
  - 기본 정책 유지(`HTTPS검증: 참`)
  - 개발 환경에서만 예외적으로 정책 완화

### 12.3 `E_TIMEOUT`
- 원인:
  - 네트워크 또는 비동기 대기 초과
- 해결:
  - `대기최대(task, ms)` 타임아웃 조정
  - HTTP 타임아웃 환경변수 조정

### 12.4 테스트는 통과하지만 실행 시 느림
- 점검:
  - `비동기통계()`로 큐 폭주 여부 확인
  - 불필요한 동기 I/O 호출 제거
  - 큰 반복문 내부 문자열/객체 변환 최소화

## 14) 구현 기준 주의사항(중요)

아래 항목은 현재 Rust 구현에서 실제로 검증된 동작 규칙입니다.

### 14.1 최상위 제어문 제한
- 함수 바깥 `반환`은 금지됩니다. (오류)
- 반복문 바깥 `중단`/`계속`은 금지됩니다. (오류)
- 모듈 최상위에서도 `반환`/`중단`/`계속`은 금지됩니다.

### 14.2 변수 재선언 규칙
- 동일 실행 환경에서 같은 이름의 변수를 다시 `변수`로 선언하면 오류입니다.
- 반복문 내부에서 매 반복마다 동일 이름 `변수`를 재선언하는 패턴은 실패할 수 있습니다.
- 권장 패턴:
  - 루프 바깥에서 1회 선언 후 루프 안에서 재할당

예시:
```cokac
변수 tmp = 없음;
동안 (변수 i = 0; i < 10; i = i + 1) {
    tmp = i;
}
```

### 14.3 모듈 경로 해석
- `모듈가져오기(path)`의 상대경로는 현재 실행 파일(또는 현재 모듈 파일) 기준으로 해석됩니다.
- 모듈 내부 함수에서 다시 `모듈가져오기(...)`를 호출하는 경우에도 호출 시점 파일 기준이 적용됩니다.
- 모듈 경로를 안정적으로 유지하려면 프로젝트 내 기준 경로를 일관되게 사용하는 것을 권장합니다.

### 14.4 클로저/모듈 경계
- 모듈이 반환한 클로저 함수는 현재 구현에서 정상 지원됩니다.
- 모듈 내부 객체 리터럴(예: 클래스 메서드 객체)의 익명 함수도 모듈 arena에 바인딩되어 실행됩니다.
- 독립 클로저 인스턴스 간 상태는 서로 간섭하지 않아야 하며, 회귀 테스트로 확인하는 것을 권장합니다.

### 14.5 테스트 설계 권장
- `--test`로 실행할 폴더에는 실제 테스트 파일(`.cokac`)만 두세요.
- 보조 모듈은 `.mod` 등으로 분리하세요.
- HTTP 로컬 테스트는 `127.0.0.1` + 고정 포트를 사용하고, 필요 시 `COKAC_HTTP_BLOCK_LOCALHOST=0`를 설정하세요.

## 17) 정식 문법 완전판(구현 매핑)

이 절은 파서/평가기의 실제 동작을 학습자 관점에서 직접 대응시키기 위한 완전판 문법입니다.

### 17.1 Top-Level
- `program := declaration* EOF`
- `declaration := varDecl | constDecl | functionDecl | statement`

### 17.2 Declarations
- `varDecl := "변수" IDENT "=" expression ";"`
- `constDecl := "상수" IDENT "=" expression ";"`
- `functionDecl := asyncOpt "함수" IDENT "(" paramListOpt ")" block`
- `asyncOpt := "비동기" | ε`
- `paramListOpt := paramList | ε`
- `paramList := IDENT ("," IDENT)*`

### 17.3 Statements
- `statement := printStmt | ifStmt | whileStmt | forStmt | returnStmt | breakStmt | continueStmt | importStmt | tryStmt | throwStmt | assignStmt | exprStmt | block`
- `block := "{" declaration* "}"`
- `printStmt := "출력" expression ";"`
- `ifStmt := "만약" "(" expression ")" block ("아니면" (ifStmt | block))?`
- `whileStmt := "반복" "(" expression ")" block`
- `forStmt := "동안" "(" forInit expressionOpt ";" forIncrementOpt ")" block`
- `forInit := varDecl | constDecl | exprStmt | ";"`
- `forIncrementOpt := forIncrement | ε`
- `forIncrement := IDENT "=" expression | expression`
- `expressionOpt := expression | ε`
- `returnStmt := "반환" expressionOpt ";"`
- `breakStmt := "중단" ";"`
- `continueStmt := "계속" ";"`
- `importStmt := "가져오기" STRING ("별칭" IDENT)? ";"`
- `tryStmt := "시도" block (catchPart)? (finallyPart)?`
- `catchPart := "잡기" "(" IDENT ("," IDENT)? ")" block`
- `finallyPart := "마침" block`
- 제약: `catchPart` 또는 `finallyPart` 중 최소 1개 필요
- `throwStmt := "던지기" expression ";"`
- `assignStmt := IDENT "=" expression ";" | memberAccess "=" expression ";" | indexAccess "=" expression ";"`
- `exprStmt := expression ";"`

### 17.4 Expressions
- `expression := logicOr`
- `logicOr := logicAnd (("또는" | "||") logicAnd)*`
- `logicAnd := equality (("그리고" | "&&") equality)*`
- `equality := comparison (("==" | "!=") comparison)*`
- `comparison := term (("<" | "<=" | ">" | ">=") term)*`
- `term := factor (("+" | "-") factor)*`
- `factor := unary (("*" | "/" | "%") unary)*`
- `unary := ("!" | "-" | "대기") unary | postfix`
- `postfix := primary (("(" argListOpt ")") | ("." IDENT) | ("[" expression "]"))*`
- `argListOpt := argList | ε`
- `argList := expression ("," expression)*`
- `memberAccess := postfix "." IDENT`
- `indexAccess := postfix "[" expression "]"`

### 17.5 Primaries
- `primary := NUMBER | STRING | "참" | "거짓" | "없음" | IDENT | "(" expression ")" | arrayLit | objectLit | anonFn`
- `arrayLit := "[" (expression ("," expression)*)? "]"`
- `objectLit := "{" ((IDENT | STRING) ":" expression ("," (IDENT | STRING) ":" expression)*)? "}"`
- `anonFn := "함수" "(" paramListOpt ")" block`

### 17.6 Lexical
- `IDENT := (LETTER | "_" | NON_ASCII) (LETTER | DIGIT | "_" | NON_ASCII)*`
- `LETTER := UnicodeAlphabetic`
- `DIGIT := [0-9]`
- `NON_ASCII := codepoint >= 0x80`
- `NUMBER := DIGIT+ ("." DIGIT+)?`
- `STRING := '"' (ESC | ~["\\])* '"'`
- `ESC := "\\" ("n" | "r" | "t" | "\\" | "\"")` (`\"`는 큰따옴표 이스케이프)

### 17.7 문법 차이 주의
- 문서/일반 언어 이론에서는 대입을 표현식에 포함하기도 하나, 현재 구현은 대입을 **문장 레벨**로 처리합니다.
- `for` 초기화는 `varDecl/constDecl/exprStmt/빈문장`을 허용합니다.
- `if/while/for` 본문은 현재 구현상 `block`(`{}`)이 필요합니다.

## 18) 연산 의미론 표(타입 조합별)

### 18.1 산술/문자열

| 연산 | 입력 타입 | 결과 | 비고 |
|---|---|---|---|
| `+` | 숫자 + 숫자 | 숫자 합 | 정상 |
| `+` | 그 외 조합 | 문자열 결합 | 양쪽 `to_display_string` 후 결합 |
| `-`, `*`, `/`, `%` | 숫자 변환 가능 | 숫자 | `숫자/불린/숫자문자열` 변환 |
| `/`, `%` | 우항 0 | 오류 | `0으로 나눌 수 없습니다.` |

### 18.2 비교

| 연산 | 규칙 |
|---|---|
| `< <= > >=` | 양쪽을 숫자로 변환 후 비교 |
| `== !=` | `Value::equals` 구조 비교 규칙 사용 |

### 18.3 동등 비교(`==`) 상세

| 타입 | 규칙 |
|---|---|
| 숫자 | `abs(a-b) < 1e-9` |
| 불린/문자열/없음 | 값 동일성 |
| 함수 | 내장함수: 이름 비교 / 사용자함수: 본문 인덱스+이름 비교 |
| 배열 | 길이 같고 각 요소 재귀 비교 |
| 객체 | 키 개수 같고 동일 키의 값 재귀 비교 |
| 작업(Task) | 완료/실패 상태 + 결과값(있으면) 비교 |

### 18.4 논리 연산

| 연산 | 평가 방식 | 반환 |
|---|---|---|
| `그리고` | 좌항 falsy면 우항 미평가 | 좌항 또는 우항 원값 |
| `또는` | 좌항 truthy면 우항 미평가 | 좌항 또는 우항 원값 |
| `!x` | `is_truthy` 반전 | 불린 |

### 18.5 진릿값 판정
- falsy: `거짓`, `없음`, 빈 문자열, `abs(n) <= 1e-9` 숫자, 빈 배열, 빈 객체, 실패/미완료 작업
- truthy: 그 외

### 18.6 런타임 오류 주요 조건
- 정의되지 않은 변수 접근/대입
- 상수 재할당
- 불변 배열/객체 수정
- 배열 인덱스 범위 초과
- 객체가 아닌 값에 속성 접근/설정
- 호출 불가 타입 호출

## 19) 스코프/클로저 시각화

### 19.1 환경 체인
- 전역: 스크립트 최상위 환경
- 함수 호출: 호출 시 새 환경 생성, 부모는 호출/캡처 환경
- 블록: 현재 구현은 별도 블록 스코프를 매번 생성하지 않고 같은 환경을 사용

### 19.2 이름 해석 순서
1. 현재 환경
2. 부모 환경
3. 상위로 반복
4. 실패 시 `E_UNDEFINED_VARIABLE`

### 19.3 클로저 캡처
- 중첩 함수 선언 시 렉시컬 환경을 캡처
- 반환된 함수가 이후 호출되어도 캡처된 부모 환경 상태를 유지

예시:
```cokac
함수 카운터생성() {
    변수 n = 0;
    함수 증가() {
        n = n + 1;
        반환 n;
    }
    반환 증가;
}

변수 c1 = 카운터생성();
출력 c1(); // 1
출력 c1(); // 2
```

### 19.4 상수/불변
- `상수 x = 값;`에서 배열/객체는 재귀적으로 freeze
- 상수 변수 자체 재할당 불가
- 불변 컨테이너의 변경 연산은 런타임 오류

## 21) 비동기/Task 상태 머신

### 21.1 상태 정의
- `대기`: 완료 전
- `완료`: 성공 완료
- `실패`: 실패 완료
- `취소`: 취소 완료(실패로 분류)

### 21.2 상태 전이
1. 생성 -> `대기`
2. 성공 완료 -> `완료`
3. 오류 완료 -> `실패`
4. 취소 요청 성공 -> `취소`
5. 완료 후 추가 전이는 없음

### 21.3 관측 함수 대응
- 상태: `작업상태`, `작업완료`, `작업실패`
- 결과: `작업결과`
- 오류: `작업오류`, `작업오류코드`
- 제어: `작업취소`

### 21.4 조합기 의미론
- `작업모두([t1..tn])`:
  - 모든 작업 완료 시 결과 배열
  - 하나라도 실패하면 실패 경로로 완료
- `작업경주([t1..tn])`:
  - 가장 먼저 완료(성공/실패)한 작업의 결과를 반영

### 21.5 타임아웃/대기
- `대기 task`: 완료까지 블록
- `대기최대(task, ms)`: 제한 시간 내 완료 대기, 초과 시 타임아웃 오류

### 21.6 Race 조건 주의
- 취소와 완료가 경합할 때 먼저 확정된 상태가 유지
- 운영 코드는 상태 조회 후 결과 접근

## 22) 오류 체계 규범화

### 22.1 표준 오류 객체

```json
{
  "메시지": "...",
  "코드": "E_RUNTIME",
  "줄": 12,
  "스택": ["함수A:10", "함수B:4"]
}
```

### 22.2 오류 분류 원칙
- 프로그램 로직 오류: `E_RUNTIME`, `E_ARGUMENT`, `E_OUT_OF_RANGE`
- 보안/정책 오류: `E_SECURITY_POLICY`, `E_NOT_ALLOWED`
- I/O/네트워크 오류: `E_HTTP*`, `E_FILE_*`, `E_FILESYSTEM`, `E_TIMEOUT`
- 비동기 오류: `E_TASK`, `E_TASK_CANCELLED`, `E_BACKPRESSURE`

### 22.3 재현/복구 전략 템플릿
- `E_TIMEOUT`: 타임아웃 상향 + 재시도(횟수 제한)
- `E_HTTP_STATUS`: 상태코드 기반 분기(4xx/5xx)
- `E_FILE_NOT_FOUND`: 경로/권한/배포 아티팩트 검증
- `E_BACKPRESSURE`: 큐 제한/동시성 낮추기

### 22.4 예외 처리 권장 패턴
```cokac
시도 {
    변수 r = 웹가져오기(url);
    반환 r;
} 잡기(메시지, 오류정보) {
    만약 (오류정보.코드 == "E_TIMEOUT") {
        반환 "재시도 필요";
    }
    던지기 메시지;
}
```

## 27) 내장함수 시그니처 사전(핵심 규범)

표기:
- `f(a, b?)`: `?`는 선택 인자
- `f(a...)`: 가변 인자
- `->`: 반환 타입
- 오류코드: 대표 코드만 기재(상황에 따라 `E_RUNTIME`로 귀결 가능)

### 27.1 Core/변환
- `길이(x) -> 숫자` (`E_ARGUMENT`)
- `단언(조건, 메시지?) -> 없음` (`E_RUNTIME`)
- `문자열(x) -> 문자열`
- `불린(x) -> 불린`
- `숫자(x) -> 숫자` (`E_ARGUMENT`)
- `절댓값(n) -> 숫자` (`E_ARGUMENT`)
- `정수(n) -> 숫자` (`E_ARGUMENT`)
- `최대(n...) -> 숫자` (`E_ARGUMENT`)
- `최소(n...) -> 숫자` (`E_ARGUMENT`)
- `타입(x) -> 문자열`
- `불변인가(x) -> 불린`

### 27.2 문자열
- `문자포함(문자열, 부분) -> 불린`
- `문자치환(문자열, 찾을값, 바꿀값) -> 문자열`
- `문자분할(문자열, 구분자) -> 배열`
- `문자시작(문자열, 접두사) -> 불린`
- `문자끝(문자열, 접미사) -> 불린`
- `문자다듬기(문자열) -> 문자열`
- `문자대문자(문자열) -> 문자열`
- `문자소문자(문자열) -> 문자열`
- `문자시작제거(문자열, 접두사) -> 문자열`
- `문자끝제거(문자열, 접미사) -> 문자열`
- `문자반복(문자열, 횟수) -> 문자열`

### 27.3 배열
- `배열추가(배열, 값) -> 배열` (`E_ARGUMENT`, 불변 시 `E_NOT_ALLOWED/E_RUNTIME`)
- `배열삽입(배열, 인덱스, 값) -> 배열` (`E_ARGUMENT`, `E_OUT_OF_RANGE`)
- `배열삭제(배열, 인덱스) -> 값` (`E_ARGUMENT`, `E_OUT_OF_RANGE`)
- `배열꺼내기(배열) -> 값|없음`
- `배열슬라이스(배열, 시작, 끝?) -> 배열`
- `배열합치기(배열, 배열) -> 배열`
- `배열정렬(배열) -> 배열`
- `배열문자열합치기(배열, 구분자) -> 문자열`
- `배열맵(배열, 함수) -> 배열`
- `배열필터(배열, 함수) -> 배열`
- `배열리듀스(배열, 함수, 초기값) -> 값`

### 27.4 객체
- `객체가짐(객체, 키) -> 불린`
- `객체설정(객체, 키, 값) -> 객체` (불변 객체 오류 가능)
- `객체삭제(객체, 키) -> 불린`
- `객체키들(객체) -> 배열`
- `객체값들(객체) -> 배열`
- `객체복사(객체) -> 객체`
- `객체합치기(대상, 소스) -> 객체`

### 27.5 파일/디렉토리/경로
- `파일읽기(경로) -> 문자열` (`E_FILE_NOT_FOUND`, `E_FILE_PERMISSION`)
- `파일읽기줄들(경로) -> 배열`
- `파일쓰기(경로, 내용) -> 불린`
- `파일쓰기줄들(경로, 배열) -> 불린`
- `파일추가(경로, 내용) -> 불린`
- `파일복사(원본, 대상) -> 불린`
- `파일이동(원본, 대상) -> 불린`
- `파일존재(경로) -> 불린`
- `파일삭제(경로) -> 불린`
- `파일정보(경로) -> 객체`
- `파일크기(경로) -> 숫자`
- `파일수정시각(경로) -> 숫자`
- `디렉토리목록(경로) -> 배열`
- `디렉토리생성(경로) -> 불린`
- `디렉토리삭제(경로) -> 불린`
- `디렉토리삭제재귀(경로) -> 불린`
- `디렉토리복사(원본, 대상) -> 불린`
- `디렉토리존재(경로) -> 불린`
- `현재디렉토리() -> 문자열`
- `경로합치기(a, b, ...) -> 문자열`
- `절대경로(경로) -> 문자열`
- `상대경로(기준, 대상) -> 문자열`
- `경로정규화(경로) -> 문자열`
- `경로존재(경로) -> 불린`
- `경로이름(경로) -> 문자열`
- `상위경로(경로) -> 문자열`
- `확장자(경로) -> 문자열`

### 27.6 JSON/인코딩/해시
- `자료파싱(문자열) -> 값` (`E_JSON`)
- `자료문자열화(값) -> 문자열` (`E_JSON`)
- `자료예쁘게문자열화(값, 들여쓰기?) -> 문자열`
- `자료읽기/자료파일읽기(경로) -> 값`
- `자료쓰기/자료파일쓰기(경로, 값) -> 불린`
- `자료예쁘게쓰기/자료파일예쁘게쓰기(경로, 값, 들여쓰기?) -> 불린`
- `해시문자열(문자열) -> 문자열`
- `해시파일(경로) -> 문자열`
- `베이스육십사인코드(문자열) -> 문자열`
- `베이스육십사디코드(문자열) -> 문자열`

### 27.7 시간/난수/입출력/프로세스
- `현재시간() -> 숫자`
- `시간문자열(타임스탬프?) -> 문자열`
- `대기밀리초(ms) -> 없음`
- `난수() -> 숫자`
- `난수정수(최소, 최대) -> 숫자`
- `입력/사용자입력(프롬프트?) -> 문자열`
- `표준입력읽기() -> 문자열`
- `표준출력쓰기(x) -> 없음`
- `표준출력줄(x?) -> 없음`
- `표준에러쓰기(x) -> 없음`
- `표준에러줄(x?) -> 없음`
- `명령실행(명령) -> 문자열` (`E_RUNTIME`, 운영 정책상 주의)
- `명령실행결과(명령) -> 객체`
- `인수값(인덱스) -> 문자열|없음`
- `인수개수() -> 숫자`
- `인수목록() -> 배열`
- `환경(키) -> 문자열|없음`
- `환경목록() -> 객체`
- `종료(코드?) -> 없음` (프로세스 종료)

### 27.8 HTTP/서버
- `웹가져오기(url, 옵션?) -> 객체` (`E_HTTP*`, `E_TIMEOUT`, `E_SECURITY_POLICY`)
- `웹요청(method, url, 헤더?, 본문?) -> 객체`
- `가져오기요청(url, 옵션?) -> 객체` (fetch 스타일)
- `응답본문(응답객체) -> 문자열`
- `응답자료(응답객체) -> 값` (`E_JSON`)
- `응답헤더값(응답객체, 키) -> 문자열|없음`
- `서버열기(port, host?, backlog?) -> 숫자(서버 FD)`
- `요청받기(서버FD) -> 객체(요청)`
- `응답보내기(연결FD, 상태코드, 본문, 헤더?) -> 없음`
- `연결닫기(FD) -> 없음`

### 27.9 Task/비동기/OOP/모듈
- `작업완료(task) -> 불린`
- `작업실패(task) -> 불린`
- `작업오류(task) -> 문자열|없음`
- `작업오류코드(task) -> 문자열|없음`
- `작업상태(task) -> 문자열`
- `작업결과(task) -> 값|없음`
- `작업취소(task) -> 불린`
- `작업모두(배열<task>) -> task`
- `작업경주(배열<task>) -> task`
- `대기최대(task, ms) -> 값` (`E_TIMEOUT`)
- `비동기큐길이() -> 숫자`
- `비동기통계() -> 객체`
- `클래스생성(이름, 메서드객체?) -> 객체`
- `인스턴스생성(클래스, 인자...) -> 객체`
- `메서드호출(인스턴스, 메서드명, 인자...) -> 값`
- `클래스확인(x) -> 불린`
- `상속확인(클래스, 상위클래스) -> 불린`
- `내보내기(이름, 값) -> 없음`
- `모듈가져오기(경로) -> 객체`

### 27.10 비동기 접두사 함수 규칙
- `비동기*` 함수는 기본적으로 동기 함수를 task로 래핑
- 별도 구현: `비동기가져오기요청`, `비동기웹요청`, `비동기요청받기`
- 나머지는 `sync_name` 매핑으로 동작(구현: `handle_async_builtin`)

### 27.11 Core/변환 예제 + 실패 예제

```cokac
출력 길이("abcd");              // 4
출력 문자열(123);               // "123"
출력 불린(0);                   // 거짓
출력 숫자("12.5");              // 12.5
출력 정수(9.9);                 // 9
```

### 27.21 함수 사용 시 공통 실패 방지 규칙
1. 외부 입력값은 함수 호출 전에 `타입`/`단언`으로 검증.
2. 파일/HTTP/비동기 호출은 항상 `시도/잡기`로 감싸고 `오류정보.코드`로 분기.
3. 운영 코드에서 `명령실행*` 사용 시 입력 허용 목록(allow-list) 적용.
4. 비동기 task는 `작업상태` 확인 후 `작업결과` 접근.

### 27.22 인자 타입 표(엄격/암묵 변환 규칙)

판정 기준:
- `Strict`: 지정 타입이 아니면 오류
- `Coerce`: 내부에서 문자열화/숫자변환/truthy 변환을 허용
- `Mixed`: 인자별로 `Strict`/`Coerce`가 섞임

#### 27.22.1 Core/변환

| 함수 | 인자 계약 | 변환 정책 | 비고 |
|---|---|---|---|
| `길이` | 문자열/배열/객체 권장 | Mixed | 기타 타입은 구현별 오류 가능 |
| `단언` | 조건(any), 메시지(선택) | Coerce | 조건은 truthy 판정 |
| `문자열` | any | Coerce | `to_display_string` |
| `불린` | any | Coerce | truthy/falsy 기준 |
| `숫자` | 숫자/불린/숫자문자열 | Coerce | 실패 시 `E_ARGUMENT` |
| `절댓값` | 숫자 계열 | Coerce | 내부 숫자변환 |
| `정수` | 숫자 계열 | Coerce | 내부 숫자변환 |
| `최대`/`최소` | 숫자들 | Coerce | 각 인자 숫자변환 |
| `타입` | any | Strict | 타입명 문자열 반환 |
| `불변인가` | any | Strict | 배열/객체 외는 `거짓` |

#### 27.22.2 문자열

| 함수군 | 인자 계약 | 변환 정책 |
|---|---|---|
| `문자포함`, `문자치환`, `문자분할`, `문자시작`, `문자끝` | 문자열 중심 | Mixed |
| `문자다듬기`, `문자대문자`, `문자소문자` | 문자열 1개 | Mixed |
| `문자시작제거`, `문자끝제거` | 문자열 2개 | Mixed |
| `문자반복` | (문자열, 횟수숫자) | Mixed (`횟수`는 Coerce) |

권장:
1. 문자열 함수 입력은 `문자열(x)`로 명시 변환 후 사용
2. 반복 횟수는 음수/과대값 방지 단언 사용

#### 27.22.3 배열/객체

| 함수군 | 인자 계약 | 변환 정책 | 실패 포인트 |
|---|---|---|---|
| `배열추가/삽입/삭제/꺼내기/슬라이스` | 1st=배열, index=숫자 | Mixed | 인덱스 범위, 불변 배열 |
| `배열합치기/정렬/문자열합치기` | 배열 | Mixed | 정렬 불가 타입 혼합 |
| `배열맵/필터/리듀스` | (배열, 함수[,초기값]) | Strict | 함수 인자 누락 |
| `객체가짐/설정/삭제/키들/값들/복사/합치기` | 객체 중심 | Mixed | 불변 객체 수정 |

#### 27.22.4 파일/디렉토리/경로

| `E_FILE_NOT_FOUND` | 경로 없음 | 파일 존재 확인 | 배포 경로 교정 |
| `E_FILE_PERMISSION` | 권한 부족 | 권한 수정 | 실행 계정 정책 점검 |
